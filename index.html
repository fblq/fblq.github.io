万历十五年有感

2019-03-12

引言

第一次接触这本书是在一本好书节目上，当初并没有留下什么印象，至今读完后，也没想明白为什么这本书会受很多读者的追捧。
我是在一次去同学大学校园里玩的机会中，同学送了这本书给我读，并叮嘱我这本书值得多读几遍，这样看来，我怕是要辜负这位好友的期望了。
趁这段时间空闲，我想记录一下我在看这本时，作者对其中人物的分析和一些零星的感想。

书中自序

中国福员广大，情形复杂。明朝采取严格的中央集权，施政方针不着眼于提倡扶助先进的经济，以增益全国财富，而是保护落后的经济，以均衡的姿态维持王朝的安全。
这种维护落后的农业经济、不愿发展商业及金融的做法，正是中国在世界范围内由先进的汉唐演变为落后的明清的主要原因。
笔者认为，中国两千年来，以道德代替法制，至明代至极，这就是一切问题的症结。写本书的目的，也重在说明这一看法。
这本《万历十五年》，意在说明16世纪中国社会的传统的历史背景，也就是尚未与世界潮流冲突时的侧面形态。有了这样一个历史的大失败，就可以保证冲突既开，恢复故态绝无可能，因之而给中国留下了一个翻天覆地、彻底创造历史的机缘。

书中人物及体会

万历十五年可以称为一个大失败的总记录，叙及的人物有万历皇帝朱翎钧，大学士张居正、申时行，南京都察院都御史海瑞，蓟州总兵官戚继光，以知府身份挂冠而去的名士李贽，他们或身败，或名裂，没有一个人功德圆满。

万历皇帝朱翎钧-青少年时期勤奋好学，励精图治；中年时期消极怠工，愚弄群臣；老年时期无为而治，就是活着的祖宗。

  • 线索：介绍万历十五年的世界大背景，以午朝小事引出礼仪约束下国家的尊卑等级->万历皇帝需要遵守的各种礼仪及其象征意义->十岁万历皇帝在学习、政务方面要做的事情，以及在生活上要受两位皇太后、张居正、冯保的监督->明朝君主制度中内阁和宦官的职能介绍，引出少年时万历对张居正和冯保的倚重和依赖->张居正长逝后，青年万历勤奋好学、励精图治，为了掌握大权发生的转变->大风起于青萍之末，拉开了清算张居正和驱逐冯保的历史帷幕->万历掌握大权后对生活的厌倦即将开始见诸行动。
  • 人物事件：万历5岁能识文读字，9岁时当了皇帝，就要成为全国臣民在礼仪等各方面的典范。各种国家大事是由首辅张居正及其领导的大学士在打理，各种奏章的批阅，万历也只是按照他们的意愿进行批奏，如“知道了”，其中为人处世的奥妙之处，万历只能随着年龄的增长而加深理解。平日倍受万历尊重的张居正从小严格监督万历的教育，并要求朝廷上下勤俭节约，可张居正被清算时，自己却奢侈无度，占尽了实利；平日平和谨慎，有君子之风的冯保却暗中玩弄权术，贻害朝廷，这些对青年万历的打击实在很大，给本朝埋下了很多政治危机。
  • 体会：皇帝的职位是一种应社会需要而产生的机构，而每个皇帝又都是一个个人。皇帝虽然是无上权威的象征，但时刻受文官集团的监察，被迫纳入了文官集团所设置规范中，不允许个性自由发展。

首辅申时行-极富才干，又能虚心下气；善于调剂矛盾，懂得阴阳调和；能认清时势，老成持稳；善用恕道待人，鼓励诚信。

  • 线索：申时行任首辅的责任及背后的意义->申时行上任首辅的机缘以及他的从政经验->帝国背景和文官集团的利弊分析，引出申时行的处世从政的方法->介绍前任张居正理想中的新政和现实的矛盾（p70） ->申时行任首辅的大背景和他的政治危机->申时行推行和前任张居正不同的国家政策以及效果。
  • 人物事件：申时行遇事行之蕴藉，半由天赋，另一半则来自从前任那里得到的教训。他认清了首辅的责任是让文官集团里的人才能在政府中发挥长处，一项政策是否能成功实施全看它是否与文官的共同习惯相安无忧。同时申时行深谙人性的弱点，没有忽略文官的双重性格：即虽称公仆，实系主人；有阳则有阴。（p65）
  • 体会：在当时明朝帝国的历史背景下。帝国高度中央集权，严重的繁文缛节禁锢了自由的思想；国家制度因循守旧，脱离实际，远远落后于时代发展的需要；处于社会中坚的文官集团，官僚作风积重难返，利益盘根错节。身为首辅，非忠烈的治世能臣，治国简直就是烫手的山芋，申时行能吸取前任首辅悲惨下场的教训，维护帝国安稳来确保自己能全始终，也是人之常情。
  • 句子摘抄：
  • 天下的大道理都可以用常情来度量。（p64）
  • 技术上的争端，一经发展，就可以升级扩大而成为道德问题，胜利者及失败者也就相应地被认为至善或至恶。（p78）
  • 经历过政治圈子里”去皮见骨”式的攻击，他们能从青萍之末，预测大风暴的来临。（p79）

世间已无张居正-聪明绝顶，能记忆千头万绪的详情末节，又极能了解各种人事的机微；自信过度，不能谦虚谨慎，不肯对事实作必要的让步。

  • 线索：明朝开始陷入宪法危机，万历皇帝消极怠工、愚弄群臣给文官集团造成的影响->十年的立储之争，文官集团中的隔阂与对立，以及文官们对万历的期望->引入万历叔祖正德皇帝的历史，分析历史现状以及万历自身性格对明朝的影响。
  • 人物事件：没有了张居正，中枢中已经没有能掌控全局的能臣。清算完张居正后的文官集团积累的隔阂与对立开始显露出来。万历对繁文缛节的意义失去了耐心，过去花费了很多时间维护的礼仪等如今已不再愿意应付，无意成为积极有为的君主，但现实无可逃遁，万历只能消极无为。从小没有体会过自由的万历缺乏勇气、积极性和寻找快乐的情趣；虽然万历懂得如何驾驭文官集团的力量，但万历缺乏坚强的毅力，孤立无援的他只好一再向臣下屈服；同时万历不是一个胸襟开阔足以容物、并以恕道待人的皇帝，他的自尊心受到损伤就设法报复。
  • 体会：道德而不是法律维系下的明朝，由于张居正对万历产生的影响，万历已经看透文官集团的虚伪，开始了消极抵抗、无为而治的“报复”行为，这对缺失法律的王朝简直就是毒药。
  • 句子摘抄：
  • 这种”阴”（即人的私欲）绝非人世间的力量所能加以消灭。（p89）
  • 文官集团积弊的根源是财政安排。（p105）

活着的祖宗-御宇后期的万历，缺乏坚强的意志和决心，但并不缺乏清醒和机灵的头脑。

  • 线索：引入申时行行政的目的、方法以及世人对他的评论->申时行的政绩以及对万历的辅佐->万历受文官集团的牵制，政治上的举措遭受各种妥协，开始心灰意冷->万历接受精神上的活埋，成了活着的祖宗。
  • 人物事件：申时行开始辅佐时的万历精神焕发，励精图治，对申时行提出的要求全力以赴，极为难得。但是在万历的行为受到文官集团的监督约束，不得不妥协后，万历开始走向了无为而治的极端。
  • 体会：我们知道的，则是万历即使有任何创造的能力，也会因为他身着龙袍，贵为天子，而无法施展表现#命运的残酷#。
  • 句子摘抄：
  • 以道德力量作为施政的根本，关键在于防止坏事的发生，而不在于琐屑地去解决问题。（p127）
  • 明朝这种特殊的政治制度之下(以德治国)，人君和人臣务必互相迁就互相让步。（p137）
  • 年轻人不知实务，轻举妄动，以致弄得事情不可收拾。（p140）
  • 道德不过是借口，问题的症结是廷臣的安全感。（p146）
  • 富有诗意的哲学家说，生命不过是一种想象，这种想象可以突破人世间的任何阻隔。（p151）

南京都察院都御史海瑞-古怪的模范官僚

  • 线索：海瑞的从政信条和执法精神->海瑞出了名的直言胆略和秉公执法，以及他经历的离奇仕途->海瑞执法的社会效应过于强烈，遭到弹劾被迫辞官->海瑞行事政令的成宪和理论依据，即洪武皇帝推行的农村政策和财政政策以及社会发展变化带来的流弊，引入海瑞执法的事例进行佐证->海瑞辞官后的生活与文集作品->海瑞重新被启用的模范作用，但文官集团对他褒贬不一。
  • 人物事件：海瑞充分重视法律的作用并且执法不阿，但是作为一个在圣经贤传培养下成长的文官，他又始终重视伦理道德的指导作用。他在著作中表示，人类的日常行为乃至一举一动，都可以根据直觉归纳于善、恶两个道德范畴内。他说，他充当地方的行政官而兼司法官，所有诉讼，十之六七，其是非可以立即判定。只有少数的案件，是非尚有待斟酌，而斟酌的标准全凭个人的判断，而不是依赖一个强有力的机构，当裁决为数众多、头绪繁多的争执时，其是否能一一做到合情合理，无疑是一个极大的疑问，从海瑞处理的高利贷下的田产纠纷、刑事案件中可以看出我们帝国的执法质量，可见国家公器太过随意。
  • 体会：在全国的广大农村中遏制了法制的成长发育，以抽象的道德代替了法律，上至官僚下至村民，其判断是否的标准是“善”和“恶”，而不是“合法”或“非法”。（p170）政府不用技术和经济的力量扶持民众，而单纯依靠政治上的压力和道德上的宣传，结果只能是事与愿违。（p172）
  • 句子摘抄：
  • 我们的帝国缺乏有效的货币制度和商业法律，这两个问题不解决，高利贷就无法避免。（p174）
  • 明朝新的税制“一条鞭法”。（p177）
  • 一个君子何以有志于做官，海瑞的回答是无非出于恻隐和义愤。他看到别人的饥寒疾苦而引起同情，同时也看到别人被损害欺压而产生不平。在君子的精神世界里，出仕做官仅仅是取得了为国家尽忠、为百姓办事的机会。一个人如果出于牟利，他可以选择别的职业，或为农，或为工，或为商。如果为仕做官，则应当排除一切利己的动机。（p184）

蓟州总兵官戚继光-孤独的将领

  • 线索：从明朝武将的遭际，引出文官和武将之间的关系->明朝落后的军事体制，遭遇倭寇横行的惨状->戚继光新军的严明纪律和训练方法->戚继光和俞大猷的军国计划对比，突出了戚继光的卓越才能->戚继光在战场上的功绩与功名->功成名遂的背后，戚继光在谭纶、张居正的襄助下，组建新军的过程->戚继光的生平事迹以及后世对他评价。
  • 人物事件：戚继光组建新军的方案考虑到了社会现实的因素，没有去触动整个国家的体制，而是脚踏实地，做职责范围内力所能及的事，同时，戚继光对军事的革新没有与传统距离过远，更不大事声张。此外，戚继光得到了谭纶的有力支持，并幸运的得到了有意重整军备的中枢大臣张居正的赏识，加上戚继光的文韬武略，才造就了辉煌功业。
  • 体会：戚继光的卓越不是在理想上把事情做得至善至美，而是最能适应环境以发挥他天才的将领。戚继光获得成功的要点，在于他清醒的现实感，他看清了并适应了当时的政治，而把军事技术作为必要的辅助，这是在当时的环境里唯一可以被允许的方案。（p214）
  • 句子摘抄：
  • 当社会和经济的发展不能平衡，冲突激化，以政治手段调剂无效，通常就会导致战争。（p193）
  • 对一个英雄人物隐恶扬善，也并不是真正的推崇。（p227）

以知府身份挂冠而去的名士李贽-自相冲突的哲学家

  • 线索：名士李贽的宿命，隐射出整个时代的悲观->李贽的家族背景及他一意孤行的原因->李贽的思想辩论及其著述->儒学的历史发展背景，引出李贽自相矛盾的学说->李贽的哲学发展之路与社会的冲突->李贽最终的不幸。
  • 人物事件：李贽历经生活的折磨后，为了追求自己的哲学理想，不顾家族反对，毅然出家以表决心。仕途出身的的他，言行毫无忌惮，与社会的矛盾冲突逐步加深，最后糟至不幸。
  • 体会：在当时的社会大背景下，文官要追求思想上个性与行动的自由需要很大的勇气，而想要创造一种思想体系去代替正统的教条则是难上加难。
  • 句子摘抄：
  • 他在重新考虑生命的意义，重建人生观之余不能再墨守成规。（p245）
  • 一个人或一种事物，其所以具有特性或功能，全靠和其他人或其他事物的相互关系。（p247）
  • 李贽的难言之隐在于他强烈地抨击了这些人物之后，他还是不得不依赖这些被抨击者的接济而生活（p252）。
  • 在李贽所处的时代，文官集团业已丧失了发展技术的可能，也没有对付新的历史问题的能力。（p264）
  • 当时政府的施政方针和个人行动完全凭借道德的指导，而它的标准又过于僵化，过于保守，过于简单，过于肤浅，和社会的实际发展不能适应。（p268）
  • 在道德的旌旗下，拘谨和雷同被视为高尚的教养，虚伪和欺诈成为官僚生活中不可分离的组成部分。（p268）

  • 商业及金融的发展依赖什么？它们是怎么促进经济发展的？
  • 社会接受思想改造需要具备什么样的条件？革命又是什么？
  • 教育和政治有什么关系呢？为什么国家要大力发展义务教育？

1.生活中看似末端小节的事情，有时会无形地塑造我们的性格，决定我们人生的走向。由于这种潜移默化的改变，难免会与现实起冲突，我们有时候就存在自我否定、自相矛盾的时候，但我们也要有正视的勇气，通过不断提升自己的智慧、拓宽自己的格局、加大思维的深度，去从容地宽恕自己并改正自己的不足。
2.内心的想法在落地变现前，中间存在一道现实的鸿沟，影响结果的因素随着局势在变化，或好或坏，自己要怀着乐观的心态期待好的结果，对坏的结果也要有提前准备。
3.越优秀的人需要承受的压力、束缚就越多，就越要懂得审时度势，越要团结身边一切可能的力量，从小事上善待身边的每一个值得信赖的人开始，推心置腹地去相互欣赏、相互学习。

拓展性摘抄

  • 中国传统封建经济为什么不能向资本主义经济发展？

    资本主义是一种组织，一种系统。即马克思在《资本论》第二卷中论述了资本主义的流通方式，其公式亦为C-M-C，即商品交换为货币，货币又再交换为商品，川流不息。但是货币是一种公众的制度，它把原来属于公众的权力授予私人。私人资本积累愈多，它操纵公众生活的权力也愈大。同时，商业资本又是工业资本的先驱，商业有了充分的发展，工业的发展才能同样地增进。这是欧美资本主义发展的特征。中国传统政治既无此组织能力，也绝不愿私人财富扩充至不易控制的地步，为王朝的安全之累。

  • 关于迷信和非迷信，其间的分野可能极其模糊。

    当一个人强迫自己对一件事情、一种前途建立信念，则其与宗教式的皈依就相去甚微。因为凡是一个人处于困境，他就不愿放弃任何足以取得成功的可能性，即使这种可能性极为渺茫，没有根据，他也要把它作为自己精神上的寄托。

产品架构

参与方

支付业务中涉及到的参与方的关系如下：

  • 名词解释：
  • 入网商户：签订服务合同中的甲方，比如针对支付服务，跟支付宝签合同的服务平台、商家、公司等。
  • 支付机构：拥有支付牌照，且配套的支付系统经过相关部门的检查，各项指标符合要求的组织，比如支付宝、微信、京东支付等。
  • 聚合服务商：没有支付牌照，但具备线上收单支付能力的组织，比如Ping++。
  • 银联、网联：国家金融服务建设的中坚，帮助支付机构对接银行的业务能力。
  • 资方：自身具备放贷资质的组织，如小额信贷公司。

例子
1.张三清空淘宝购物车的时候，点击立即支付，淘宝会以支付宝入网商户的身份，调用支付宝的收银能力，这时用户就能在支付宝里完成支付。
其中的支付宝就是一家支付机构，它以商户的身份在银联、网联注册入网，调用银联、网联的支付能力。
而银联、网联是帮助所有支付机构，如支付宝、微信、京东支付、美团支付等等，去对接所有的银行，集中把银行的支付能力转接给支付机构，各家支付机构不用挨家挨户地去找银行进行对接。
2.用户在携程APP上订完酒店后支付时，看到的支付宝、微信支付方式，就是携程转接了支付机构的支付能力。
3.对于支付机构和聚合服务商直接对接资方的场景一般是小额信贷，有蚂蚁花呗、京东白条、去哪儿拿去花、滴滴信用付等。

产品层级关系

支付业务中业务层、接入层、收单层、支付层、渠道层、外部合作方间的层级关系如下：

  • 业务：收银台（二维码、闪付、小额免密）、API服务(开户、分账、出金入金)、外部商户、开放平台
  • 接入：用户门户、商户门户、api、H5/SDK
  • 收单: 下单、构建收银台、充值、转账、提现、收款、退款、付款、查询
  • 支付：收款、退款、付款、查询、回调
  • 渠道：实名、签约、协议支付、商业委托支付、网关支付、认证支付、退款、付款、差错
  • 外部合作方：银联、网联、支付机构、商业银行

产品work机制：
1.先基于用户门户、商户门户积累的用户资源，在任何一个需要支付的页面入口处，通过一些营销手段引导用户开通支付功能，即开通支付账户并绑定银行卡，积累了这些账户资源后，就可以开始在线上开展收单支付业务了。
2.通过接入层和收单层的能力在线上服务消费的业务链终端开展收单业务，用户选择服务消费、下单、进入收银台、选择支付方式发起支付，支付层就能结合客户的开户信息及账户信息，确定商户的手续费和结算方式，同时能根据支付方式信息（银行卡、余额、营销）路由出支付通道及相应的通道手续费，向渠道层发起支付，渠道层根据选出的支付通道按照格式要求组装报文，将支付请求发往网联、银联。
3.支付层会对交易凭证数据进行持久化，涉及的核心数据有交易双方的用户信息、支付产品信息、交易订单信息、银行卡信息、通道信息、商户的结算信息，用于上报结算时给清结算使用。

信息流和资金流

用户、服务提供商有自己的银行账户和支付账户，支付机构有自己的对公账户和备付金账户，先看一下支付公司、网联、央行、银行这四个角色在交易信息流和资金流上的交互关系：（其中备付金为支付公司在央行开的备付金集中存管账户）

类比滴滴的打车场景（乘客、司机、网约车平台、滴滴支付），说一下新零售场景（用户、供应商、新零售商、支付机构），看一下支付公司怎么能帮助平台型小企业做生意。

1.用户支付（支付公司向新零售商收取手续费）
资金流：用户银行卡（-100）->支付公司的备付金账户(+100)
支付公司清结算：
信息流：待清算账户（计算手续费）->待结算账户(资金确认)->现金账户（供应商+80：新零售商+20）
2.新零售商提现
信息流：现金账户（-80）
资金流：备付金（-80）->新零售商的银行账户（+80）
3.用户充值
信息流：用户支付账户的现金账户（+100）
资金流：用户银行卡（-100）->支付公司的备付金账户（+100）
4.用户跨行转账（建行->招行）
资金流：用户建行卡（-100）->支付公司的备付金账户（+100） AND 支付公司的备付金账户（-100）->用户招行卡（+100）
收取手续费
信息流：平台手续费（+1）
资金流：用户建行卡（-1）->支付公司的备付金账户（+1）

  • 通道：单家银行的收付款能力是一个通道。

收单层向业务层输出支付能力，提供的核心接口有下单接口、收银台接口、支付接口、结果查询接口，给交易核心层屏蔽业务差异。
交易核心层向收单层提供收款、退款、付款的能力，同时留存交易凭证信息供清结算系统使用，执行具体的业务时会检查账户状态、上报风控、进行路由选择通道及确定手续费、结算方式。

支付安全

数据安全

  • 接口数据安全：接口数据的签名、利用数字信封对敏感信息进行对称加解密、对接口数据进行非对称加解密。
  • 日志数据安全：对手机号、身份证号、银行卡号、支付账号等信息进行脱敏。
  • mysql机房数据安全：独立机房，访问入口有严格的ACL白名单机制。

账户安全

用户在支付机构开一个支付账户，可以用于充值、转账、提现、消费、理财、保险、信贷等，基本能渗透到生活中的方方面面，支付使用的频次很高，因此安全性很重要。

  • 一个支付账户允许用户管理多张银行卡，用户每次支付时可以不用输入银行卡号就可以支付，支付方便的同时，账户信息是隐藏在用户信息后面，不会直接暴露在使用场景中。
  • 需要用户开通支付账户的产品是快捷支付，用户在用快捷支付时，支付机构跟渠道交互是通过协议号进行的，即使渠道请求被中间人攻击，最后拿到的也不是用户真实的银行账户信息。

资金安全

  • 入账、出账操作的事务性。
  • 对账户进行分门别类，可以对资金进行专款专用。

  • 弱引用：继承WeakReference的类被new出来的对象，是弱引用对象；当JVM发生GC时，无论内存是否足够，都会回收只被弱引用关联的对象；弱引用对象在被JVM回收后也会被存放在与之关联的引用队列里。

Spring Web MVC简介

Spring Web MVC是一种基于请求-响应模型，使用MVC架构模式思想的轻量级Web框架，能够简化WEB开发。

Spring Web MVC优势

  1. 与Spring框架集成（AOP、IOC）
  2. 约定大于配置的契约式编程
  3. 简单的Web层单元测试
  4. 灵活地URL到页面控制器的映射
  5. 模型数据通过Map数据结构实现，容易与其他视图技术集成
  6. 强大的JSP标签库
  7. 灵活的本地化、主题解析
  8. 简单的异常处理

Spring Web MVC架构

请求-响应模型

Spring Web MVC是一个基于请求驱动的Web框架，使用了前端控制器模式进行设计，根据请求映射规则分发给相应的页面控制器进行处理，请求-响应模型如下图所示。

具体执行步骤如下：

  1. 首先用户发送请求—>前端控制器，前端控制器根据请求信息（如URL）来决定选择哪一个页面控制器进行处理并把请求委托给它，即以前的控制器的控制逻辑部分；图中的1、2步骤；
  2. 页面控制器接收到请求后，进行功能处理，首先需要收集和绑定请求参数到一个对象，这个对象在Spring Web MVC中叫命令对象，并进行验证，然后将命令对象委托给业务对象进行处理；处理完毕后返回一个ModelAndView（模型数据和逻辑视图名）；图中的3、4、5步骤；
  3. 前端控制器收回控制权，然后根据返回的逻辑视图名，选择相应的视图进行渲染，并把模型数据传入以便视图渲染；图中的步骤6、7；
  4. 前端控制器再次收回控制权，将响应返回给用户，图中的步骤8；至此整个结束。

问题：

  1. 请求如何给前端控制器？（web.xml）
  2. 前端控制器如何根据请求信息选择页面控制器进行功能处理？（handlerMapping）
  3. 如何支持多种页面控制器呢？（handlerAdapter）
  4. 如何页面控制器如何使用业务对象？（springIoc依赖注入）
  5. 页面控制器如何返回模型数据？（ModelAndView）
  6. 前端控制器如何根据页面控制器返回的逻辑视图名选择具体的视图进行渲染？（ViewResolver）
  7. 不同的视图技术如何使用相应的模型数据？（map）

MVC核心架构

Spring Web MVC核心架构图，如下图所示。

核心架构的具体流程步骤如下：

  1. 首先用户发送请求——>DispatcherServlet，前端控制器收到请求后自己不进行处理，而是委托给其他的解析器进行处理，作为统一访问点，进行全局的流程控制；
  2. DispatcherServlet——>HandlerMapping， HandlerMapping将会把请求映射为HandlerExecutionChain对象（包含一个Handler处理器（页面控制器）对象、多个HandlerInterceptor拦截器）对象，通过这种策略模式，很容易添加新的映射策略；
  3. DispatcherServlet——>HandlerAdapter，HandlerAdapter将会把处理器包装为适配器，从而支持多种类型的处理器，即适配器设计模式的应用，从而很容易支持很多类型的处理器；
  4. HandlerAdapter——>处理器功能处理方法的调用，HandlerAdapter将会根据适配的结果调用真正的处理器的功能处理方法，完成功能处理；并返回一个ModelAndView对象（包含模型数据、逻辑视图名）；
  5. ModelAndView的逻辑视图名——> ViewResolver， ViewResolver将把逻辑视图名解析为具体的View，通过这种策略模式，很容易更换其他视图技术；
  6. View——>渲染，View会根据传进来的Model模型数据进行渲染，此处的Model实际是一个Map数据结构，因此很容易支持其他视图技术；
  7. 返回控制权给DispatcherServlet，由DispatcherServlet返回响应给用户，到此一个流程结束。
    在此，我们可以看出具体的核心开发步骤：
  8. DispatcherServlet在web.xml中的部署描述，从而拦截请求到Spring Web MVC。
  9. HandlerMapping的配置，从而将请求映射到处理器。
  10. HandlerAdapter的配置，从而支持多种类型的处理器。
  11. ViewResolver的配置，从而将逻辑视图名解析为具体视图技术。
  12. 处理器（页面控制器）的配置，从而进行功能处理。
